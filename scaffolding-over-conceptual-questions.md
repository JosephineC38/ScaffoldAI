# Why ScaffoldAI Socratic-Scaffolds Simple Conceptual Questions

## Summary

`pass_one` correctly classifies direct conceptual questions as `CONCEPTUAL`, but `pass_two` never reads that classification. Every response is generated by the same prompt and the same system persona ("a socratic reasoning scaffold"), so the model scaffolds every answer regardless of what pass one diagnosed. The fix is to branch `pass_two`'s behavior on `classification`, not to tweak wording.

## Test Setup

Ran `prototype/architecture/two_pass_engine.py` four times, swapping the `input` variable each run (model: `gpt-4o-mini`, temperature 0.7 for pass two). Three questions were pure conceptual asks; one was an execution-error (`IPS`) question used as a control.

## Results

### Test 1 — "What is the first law of thermodynamics?"

Pass one:
```json
{"classification": "CONCEPTUAL", "reasoning_gap": "The student is asking for a definition of the first law of thermodynamics without providing any context or application.", "misconception": null}
```

Pass two:
> "That's a great question! The first law of thermodynamics is fundamental to understanding energy interactions. It essentially deals with the principle of conservation of energy. To help clarify this concept, can you think of a situation where energy is transferred or transformed? ... Let's explore this together!"

Never states the law. Redirects to a Socratic question.

### Test 2 — "What is entropy?"

Pass one:
```json
{"classification": "CONCEPTUAL", "reasoning_gap": "The student is asking for a definition of entropy without context or application.", "misconception": null}
```

Pass two:
> "That's a great question! Entropy is a fundamental concept ... but its meaning can vary depending on context. To help clarify, can you think about a situation ... where you might have encountered disorder or randomness? ... Let's explore that further!"

Same pattern: correctly diagnosed as a direct definition request, still deflected into a question instead of answering.

### Test 3 — "What is the equation for the Carnot efficiency?"

Pass one:
```json
{"classification": "CONCEPTUAL", "reasoning_gap": "The student is asking for the definition of the Carnot efficiency equation without any context or application.", "misconception": null}
```

Pass two:
> "That's a great question! ... Before diving into the equation, let's think about what the Carnot cycle represents. Can you tell me what you know about the Carnot cycle...?"

Worth noting: this is a request for a formula, not even a conceptual explanation — an unambiguous case where scaffolding adds friction rather than value. Still deflected.

### Test 4 (control) — IPS execution-error question

Input: *"I calculated the work as -500 J but I think it should be positive since the gas is doing work on the surroundings. What am I doing wrong?"*

Pass one:
```json
{"classification": "IPS", "reasoning_gap": "The student is confused about the sign convention for work done by the gas during expansion.", "misconception": "The student believes work done by the gas should always be positive, not recognizing the sign convention."}
```

Pass two responded with a guided, question-driven walkthrough of sign conventions — appropriate here, since the student has a specific misconception to work through.

**Takeaway:** pass one's classifier is accurate. The problem is entirely in pass two.

## Root Cause

Two things compound:

1. **The system prompt bakes in a fixed identity.** `system_prompt_components.json`'s `role` component says: *"You are ScaffoldAI, a socratic reasoning scaffold for undergraduate engineering thermodynamics."* This identity is injected into every pass-two call regardless of classification, so the model's default posture is "ask a guiding question," full stop.

2. **`pass_two_prompt` doesn't branch on `classification`.** Look at `two_pass_engine.py:61-69` — the prompt template is identical no matter what pass one decided:
   ```python
   pass_two_prompt = f"""
       topic: {topic}
       diagnoistic context (do not reveal this to the student): {pass_one_diagnosis}
       original student input message: {user_input}

       Using the diagnostic context above to inform your response, generate a response to help guide the student.

       Do not reveal the diagnosis or the correct answer.
   """
   ```
   `pass_one_diagnosis` does contain `classification`, but it's buried in an opaque JSON blob with no instruction telling the model that `CONCEPTUAL` should trigger fundamentally different behavior (answer directly) versus `IPS`/`IRL` (scaffold). An LLM given a strong persona ("you are a socratic scaffold") and a vague aside ("use this diagnostic to inform your response") will default to persona behavior. It also explicitly says "do not reveal ... the correct answer" — which is actively wrong for CONCEPTUAL questions, where the "correct answer" is the entire ask.

This is a classic two-pass architecture failure mode: pass one produces a decision, but pass two treats it as flavor text instead of a control signal.

## Recommended Fix

Don't rely on prompt wording alone to suppress a strong persona — branch the code path.

1. **Branch in `pass_two` on `classification` before building the prompt.** Parse `pass_one_diagnosis` back into a dict (you already have `topic` popped separately — do the same with `classification`), and use it to select between two distinct prompt templates:
   - `CONCEPTUAL` → an "answer directly" prompt: *"The student is asking a direct factual/definitional question. Give a clear, correct, concise answer (definition, law statement, or formula as appropriate). Do not respond with a question. Optionally follow the answer with one short prompt inviting them to apply it, but the direct answer must come first."*
   - `IPS` / `IRL` → keep the current socratic-guidance prompt.

2. **Stop blanket-forbidding the answer.** The instruction "Do not reveal the diagnosis or the correct answer" should only apply to `IPS`/`IRL` cases where the "correct answer" would spoil the problem the student is working. For `CONCEPTUAL`, invert it — the answer *is* the deliverable.

3. **Soften the system-prompt identity so it doesn't override the branch.** Update the `role` component in `system_prompt_components.json` from "a socratic reasoning scaffold" to something like: *"an adaptive thermodynamics tutor that answers direct factual questions plainly and uses Socratic guidance only when a student is working through their own problem or misconception."* A persona claim that strongly worded will otherwise fight against per-call instructions, even when pass two's prompt tries to override it — system-level identity tends to dominate temperature-0.7 generations.

4. **Add a lightweight eval/regression test** using the four prompts above (or similar) that asserts, post-fix: for `CONCEPTUAL` inputs, the pass-two response does not end in a question mark / does not contain scaffolding phrases like "can you think about," "let's explore," "what do you think." This is cheap to check and will catch regressions if the persona prompt drifts back toward always-scaffold behavior.

5. Longer term, consider making `pass_two` take an explicit `mode` argument (`"direct"` vs `"socratic"`) derived from classification, rather than passing the whole diagnosis blob into one prompt and hoping the model infers the right mode. Explicit control flow is more reliable than persuasion-by-prompt for a behavior this binary.

## Implementation Status

Items 1-4 above have been implemented:

- **`pass_two` now branches on `classification`** (`two_pass_engine.py:61-82`). It parses `pass_one_diagnosis` back into a dict and picks between a direct-answer prompt (`CONCEPTUAL`) and the original socratic-guidance prompt (`IPS`/`IRL`).
- **The "don't reveal the answer" instruction is now scoped to the non-`CONCEPTUAL` branch only.** The `CONCEPTUAL` prompt explicitly instructs the model to state the answer directly and *not* end the response with a question — an earlier draft allowed an "optional" trailing question, but that let the model tack on a deflecting question after the answer anyway, so it was tightened to require a statement instead (e.g. "Let me know if you'd like to see this applied to a problem.").
- **The `role` component in `system_prompt_components.json` was reworded** from a fixed "socratic reasoning scaffold" identity to an adaptive-tutor identity that explicitly names both behaviors (direct answers for factual asks, socratic guidance when a student is working through their own attempt).
- **A regression script was added at `prototype/eval/regression_conceptual.py`.** It re-runs the three conceptual prompts and the IPS control prompt from this doc, and asserts: classification is `CONCEPTUAL`, the response doesn't end in a question mark, and the response doesn't contain scaffold-y phrases (`"can you think"`, `"let's explore"`, etc.). Run it with `python3 prototype/eval/regression_conceptual.py` from anywhere in the repo — it adds `prototype/architecture/` to `sys.path` itself and exits non-zero if any check fails.

Verified post-fix output for all four test prompts — all four checks pass. Example (previously-failing case):

> **Q: "What is the first law of thermodynamics?"**
> "The first law of thermodynamics states that energy cannot be created or destroyed, only converted from one form to another. In mathematical terms, it is often expressed as ΔU = Q - W, where ΔU is the change in internal energy of a system, Q is the heat added to the system, and W is the work done by the system. This law emphasizes the conservation of energy within thermodynamic processes. Let me know if you'd like to see this applied to a problem."

The `IPS` control case still gets a fully socratic response, confirming the branch didn't regress the existing scaffolding behavior for cases where it's appropriate.
